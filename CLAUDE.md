# CLAUDE.md — Project Context for Claude Code

## What this project is

A Flask web application for browsing arXiv papers in combinatorics (math.CO).
Live at **https://arxiv.symmetricfunctions.com**.

## Tech stack

- **Backend:** Python 3 / Flask 3.0, MariaDB (PyMySQL), Gunicorn
- **Frontend:** Plain HTML/CSS/JS, Jinja2 templates, KaTeX for math rendering
- **Auth:** ORCID OAuth via Authlib
- **External APIs:** arXiv API (paper fetching), Crossref API (DOI discovery)
- **Hosting:** Shared hosting with Passenger WSGI (cPanel)

## Local development

```bash
source activate_venv.sh          # sets up ~/.cache/arxiv-venv-<hash>/
cd src && python3 app.py          # http://localhost:5000
```

Virtual environment is stored at `~/.cache/arxiv-venv-*` (not in repo). Setup script: `setup_venv.sh`.

## Configuration

All secrets are in `.env` (gitignored). Template: `.env.example`.
Production secrets: `.env.production` (gitignored, used by `pull_prod_db.sh` and `sync_to_prod.sh`).

Key env vars:
- `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME` (`arxiv_frontend`), `DB_CHARSET` (`utf8mb4`)
- `FLASK_SECRET_KEY`, `FLASK_DEBUG`
- `ADMIN_PASSWORD` — for `/admin` login
- `ADMIN_ORCID` — ORCID iD auto-granted admin on login
- `ORCID_CLIENT_ID`, `ORCID_CLIENT_SECRET` — ORCID OAuth
- `FETCH_SECRET` — key for `/fetch?key=...` endpoint

Config loader: `src/config.py`.

## Database

- **Engine:** MariaDB, database name `arxiv_frontend`, user `arxiv_user`
- **Schema:** `database/schema.sql` (fresh install only — **drops all tables**)
- **Migrations:** `database/migrate_*.sql` (idempotent, safe to re-run)
- **Pull prod → local:** `cd database && ./pull_prod_db.sh` (reads `.env.production` for SSH creds)
- **Reset local:** `database/reset_database.sh`

### Key tables

| Table | Purpose |
|-------|---------|
| `papers` | Core paper metadata (arxiv_id, title, abstract, dates, doi, doi_status, doi_confidence, doi_checked_at) |
| `authors` | Deduplicated author names with URL slugs |
| `paper_authors` | Many-to-many with author ordering |
| `paper_categories` | arXiv subject categories per paper |
| `keywords` | Curated keyword phrases (phrase, score 1-10, url, active) |
| `keyword_aliases` | Synonym mappings to keywords |
| `paper_keywords` | Auto-generated by `auto_tag.py` |
| `doi_candidates` | Crossref DOI lookup staging (pending/approved/rejected) |
| `users` | ORCID-authenticated users |
| `user_lists` | Papers saved to named lists |
| `user_watched_keywords` / `user_watched_authors` | Powers "My Feed" |
| `ignored_candidates` / `math_words` | Keyword triage metadata |
| `tags` / `paper_tags` | Legacy MSC-code system — **dead, no code references them**. Slated for removal (see TODO.md) |

### doi_status ENUM values

- `NULL` — no DOI, eligible for lookup
- `'arxiv'` — DOI came from arXiv metadata
- `'auto'` — auto-approved from Crossref (confidence >= 0.95)
- `'verified'` — admin-approved
- `'skipped'` — marked as unlikely to ever get a DOI; excluded from lookups

## Source code layout

### Flask app (`src/`)

| File | Role |
|------|------|
| `app.py` | Main Flask app — all public routes, template filters, index cache |
| `admin.py` | Admin blueprint (`/admin/*`) — keyword triage, DOI management, retag, fetch, users, symcat |
| `auth.py` | ORCID OAuth blueprint — login/logout/callback |
| `lists.py` | User lists blueprint — save/remove papers, manage list categories |
| `watch.py` | Watch blueprint — follow keywords/authors, "My Feed" |
| `config.py` | Loads `.env` → Python constants |
| `db.py` | Shared DB helpers: `get_db_connection()`, `require_user()`, `attach_authors()`, `attach_keywords()` |
| `utils.py` | Shared utilities: `slugify()`, `arxiv2bib()`, BibTeX generation |

### CLI scripts (`src/`)

| Script | What it does |
|--------|-------------|
| `fetch_arxiv.py` | Fetch papers from arXiv API (`--recent`, `--backfill`, `--arxiv-id`) |
| `auto_tag.py` | Apply keyword tags to papers (`--days`, `--since`, `--all`) |
| `extract_keywords.py` | Scan abstracts → `keywords.csv` for admin triage |
| `doi_lookup.py` | Find DOIs via Crossref (`--batch`, `--from-date`, `--to-date`, `--auto-approve`) |
| `bib_doi_backfill.py` | Bulk DOI lookup from a .bib file |

### Shell scripts (project root)

| Script | What it does |
|--------|-------------|
| `sync_to_prod.sh` | Deploy code + sync keyword tables to production (flags: `--no-tag`, `--no-code`, `--all`) |
| `fetch_papers.sh` | Wrapper: activate venv + run `fetch_arxiv.py --recent` |
| `tag_papers.sh` | Wrapper: activate venv + run `auto_tag.py` |
| `find_dois.sh` | Wrapper: activate venv + run `doi_lookup.py` |
| `database/pull_prod_db.sh` | SSH dump prod DB → import locally |

### Templates (`src/templates/`)

- `base.html` — base layout with KaTeX, Ko-fi widget, nav
- `_macros.html` — Jinja2 macros (`kw_tag_link`, etc.)
- `admin/_nav.html` — admin tab bar (Candidates, Keywords, Retag, Fetch, SymCat, DOIs, Users)
- Admin pages use a help-btn/help-popup pattern for inline documentation

## Conventions

- All DB queries use PyMySQL with `DictCursor` (results are dicts, not tuples)
- Admin AJAX endpoints return `{'ok': True, ...}` or `{'ok': False, 'error': '...'}`
- CSRF protection via Flask-WTF; JS reads `meta[name="csrf-token"]`
- URL slugs for authors are generated by `utils.slugify()`
- Keywords use lowercase phrases; aliases map to a canonical keyword
- `sync_to_prod.sh` never uploads `schema.sql` (destructive) — only `migrate_*.sql` files listed in its `MIGRATIONS` array
