How the keyword pipeline works

  1. Extract candidates — run src/extract_keywords.py locally. It scans every paper title + abstract, strips LaTeX, singularizes words, and counts how many papers each 1–4 word phrase appears in. Result: keywords.csv (≈49k phrases).
  2. Triage here — this page reads keywords.csv and lets you classify each phrase:
    • useful — specific enough to tag papers (e.g. schur function). Saved to the keywords DB table.
    • math — valid math but too broad alone (e.g. graph, polynomial). Recorded but not used for tagging.
    • ignore — boilerplate, stopword, or typo. Recorded so it stays out of the unreviewed queue.
  3. Tag papers — src/auto_tag.py loads all useful keywords (plus any aliases you've added) from the DB, tokenizes each paper the same way, and writes matches to paper_keywords. Runs automatically after each fetch, or manually via Retag.
  4. Refresh candidates — re-run extract_keywords.py periodically (e.g. after many new papers arrive) to update phrase counts and surface new candidates. Deploy the new CSV to update this page.
